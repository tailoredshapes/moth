Moth

A git-style, file-based issue tracker. Issues are markdown files in your repository. No database, no external service.

GitHub Documentation

Overview

Why Moth

Issues stored as plain markdown files, organised by status. Edit them directly or use the CLI. Everything is plain text, version-controlled alongside your code.

File-based storage

Issues live in .moth/{status}/ as markdown files. No database to manage, no service to run.

Git-native

Plain text, designed for version control. Track issue changes through git history. Commit hooks auto-prefix messages with the current issue ID.

Simple workflow

Move issues through customisable statuses with single commands. Partial ID matching—no need to type the full identifier.

Priority and severity

Track severity (crit, high, med, low) and ordering priority. Compact renumbering keeps the backlog tidy.

Usage

Quick start

# Initialise moth in your project
moth init

# Create a new issue
moth new "Fix login bug" -p high

# List issues
moth ls

# Start working on an issue
moth start x7k2m

# Mark as done
moth done x7k2m

# Show issue details
moth show x7k2m

Structure

How it works

Moth creates a directory structure in your project. Each issue is a markdown file named [id]-[priority]-[title-slug].md.

.moth/
├── config.yml
├── ready/
│   ├── x7k2m-high-fix-login-bug.md
│   └── p3j9n-med-add-dark-mode.md
├── doing/
│   └── a9f4k-crit-security-patch.md
└── done/
    └── b2h8l-low-update-docs.md

Philosophy

On story points

Moth tracks completed issue count rather than story points. Research indicates that point estimates have significant variance and often fail to predict delivery timelines. Issue count is a more reliable metric for measuring team capacity and velocity.
